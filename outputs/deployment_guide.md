# Deployment Guide: Donald Smith Ogono Portfolio

This guide outlines the steps to deploy the portfolio to **Vercel** (preferred) or **Netlify**.

## Prerequisites
- A GitHub, GitLab, or Bitbucket account.
- The `outputs/` folder content merged into a standard Next.js project structure.

## 1. Project Initialization
If you haven't already, create a new Next.js project and move the files from `/outputs` into it:

```bash
npx create-next-app@latest my-portfolio --js --tailwind --eslint
# Follow the prompts (select App Router, etc.)

# Then copy the files:
# outputs/app/ -> my-portfolio/app/
# outputs/components/ -> my-portfolio/components/
# outputs/data/ -> my-portfolio/data/
```

Ensure the following dependencies are installed:
```bash
npm install framer-motion
```

## 2. Local Verification
Run the development server to verify everything looks correct:
```bash
npm run dev
```

## 3. Deploy to Vercel (Recommended)
1. **Push to GitHub**: Create a new repository and push your project.
2. **Import to Vercel**: 
   - Go to [vercel.com](https://vercel.com).
   - Click "Add New" -> "Project".
   - Import your repository.
3. **Configure**: Vercel will automatically detect Next.js.
4. **Deploy**: Click "Deploy". Your site will be live on a `*.vercel.app` domain.

## 4. Security Notes
- The portfolio includes a simulation of a secure contact form. For real functionality, consider integrating with a service like **Formspree** or a custom **Next.js API Route**.
- The `Content-Security-Policy` and other headers are currently implemented via meta tags in `layout.js`. For production, it is recommended to move these to a `next.config.js` or `middleware.js` file for full header support.

## 5. Domain Setup
Once deployed, you can add your custom domain (e.g., `donaldsmith.dev`) in the Vercel Dashboard under **Settings -> Domains**.
